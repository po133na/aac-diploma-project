# Unim – iOS AAC приложение (дипломный проект)

## 📱 Что это?
Мобильное приложение для генерации учебных карточек с изображениями и озвучкой (AAC – Augmentative and Alternative Communication).  
Фронтенд: iOS (SwiftUI), бэкенд: Python (FastAPI).

## 🏗️ Архитектура
- **Фронтенд:** SwiftUI, MVVM, async/await
- **Бэкенд:** FastAPI, SQLite, Hugging Face (генерация изображений), edge-tts (озвучка)
- **Аутентификация:** JWT (Bearer токены)

## 🚀 Быстрый старт

### 1. Запуск бэкенда (уже готов!)
Бэкенд **уже запущен на сервере** и доступен по адресу:  
`http://localhost:8000` (если фронтенд работает на том же сервере)  
или  
`http://<IP_сервера>:8000` (если симулятор на другом компьютере).

**Статус бэкенда:**
```bash
curl http://localhost:8000/
→ {"status":"running","service":"AAC Image Generator","version":"2.0.0"}
```

### 2. Открой проект в Xcode
```bash
cd /root/frontend
open diploma2.xcodeproj
```
(Или перетащи `diploma2.xcodeproj` в Dock Xcode.)

### 3. Настройка симулятора
- **Xcode → Product → Destination** → выбери симулятор (iPhone 15, iOS 17+)
- Если симулятор запускается на **другом компьютере** (не на сервере), измени базовый URL в коде:
  
  **Файл:** `diploma2/Services/Network/APIClient.swift`  
  **Найди строку:** `baseURL = "http://localhost:8000"`  
  **Замени на:** `baseURL = "http://<IP_сервера>:8000"`

- **Важно:** для HTTP (не HTTPS) запросов нужно добавить в `Info.plist`:
  ```xml
  <key>NSAppTransportSecurity</key>
  <dict>
      <key>NSAllowsArbitraryLoads</key>
      <true/>
  </dict>
  ```

### 4. Запуск приложения
- **Xcode → Product → Run (⌘R)**
- Дождись сборки и запуска симулятора.

## 🔐 Первый вход

### Тестовый пользователь
```
Email:    test@example.com
Password: password123
```
При первом запуске **нужно войти через LoginView** (если нет автоматического входа).

### Как войти:
1. Открой вкладку **Settings** (шестерёнка) → нажми **Login** (если есть).
2. Введи email и пароль выше.
3. После успешного входа токен сохранится в `UserDefaults` и будет автоматически подставляться в заголовки запросов.

## 🧪 Тестирование функций

### 1. Создание карточки
- **Главный экран (Home)** → нажми **+** (большая круглая кнопка внизу).
- **Шаг 1:** выбери "Generate with AI".
- **Шаг 2:** введи слово (например, "яблоко") → нажми **Generate Image**.
- **Шаг 3:** превью изображения → **Сохранить и продолжить** (или "Сгенерировать заново").
- **Шаг 4:** введи имя карточки → **Save Card**.

### 2. Просмотр карточек (Галерея)
- Перейди на вкладку **Gallery** (иконка фото).
- **Все карточки** – сетка созданных карточек с изображениями.
- **Избранное** – карточки с ♥️.
- **Категории** – папки для организации.

### 3. Взаимодействие с карточками
- **♥️** – добавить/убрать из избранного.
- **🔊** – озвучить слово (TTS).
- **↻** в заголовке – обновить список.

### 4. TTS (озвучка)
- Автоматически работает при нажатии 🔊.
- Использует бэкенд (`/tts`) или системный синтезатор (fallback).

## ⚠️ Важные заметки

### Токен Hugging Face
- **Сейчас используется заглушка** – изображения генерируются как цветные квадраты.
- **Чтобы включить реальную генерацию:** предоставь `HUGGINGFACE_API_TOKEN` в файл `/root/backend/.env`.
- Пока токена нет – всё работает, но изображения будут простыми цветными заливками.

### Аутентификация
- **Bearer токен обязателен** для запросов к `/cards`, `/cards/generate`, `/tts`.
- После логина токен сохраняется автоматически.
- Если карточки не загружаются – проверь, что ты вошёл в аккаунт (или перезапусти приложение).

### Известные "баги"
- **GalleryView раньше был заглушкой** – теперь полностью реализован.
- **Кнопки должны "тыкаться"** – навигация между шагами создания карточки исправлена.
- **Если изображения не показываются** – возможно, `image_base64` повреждён; бэкенд всегда возвращает валидную fallback‑картинку.

## 🐛 Отладка

### Логи фронтенда
- **Xcode → Console (⇧⌘C)** – смотри логи `[APIClient]` и `[GalleryViewModel]`.
- **Проверь токен:** `[APIClient] Using token: eyJhbGciOi...` (должен быть после логина).

### Проверка бэкенда
```bash
# Статус
curl http://localhost:8000/

# Логи
tail -f /root/backend/image_generator/server5.log
```

### Ручное тестирование API
```bash
# Логин (получить токен)
curl -X POST http://localhost:8000/auth/login \
  -d '{"email":"test@example.com","password":"password123"}'

# Создать карточку (с токеном)
curl -H "Authorization: Bearer <token>" \
  -X POST http://localhost:8000/cards \
  -d '{"word":"яблоко","language":"ru"}'

# Получить все карточки
curl -H "Authorization: Bearer <token>" http://localhost:8000/cards
```
